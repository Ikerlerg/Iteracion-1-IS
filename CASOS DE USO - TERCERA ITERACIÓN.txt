CASOS DE USO - TERCERA ITERACIÓN 

1. NUEVOS CASOS DE USO PROPUESTOS:

CASO DE USO 1: Visualizar reseñas de vendedor
Clase de dominio principal: Comprador
Descripción: El comprador visualiza el perfil de un vendedor específico para consultar una lista con todas las valoraciones y comentarios (Reviews) que otros usuarios le han dejado previamente, ayudándole a decidir si realizar una compra u oferta.

CASO DE USO 2: Modo Invitado
Clase de dominio principal: Guest
Descripción: Un usuario no registrado navega por el catálogo para acceder a las listas de los productos disponibles, pero la interfaz gráfica oculta o bloquea los botones de compra, realizar oferta y publicar valoración hasta que inicie sesión.

CASO DE USO 3: Eliminar cuenta
Clase de dominio principal: Usuario
Descripción: El usuario accede a su cuenta y se introduce a su menú propio para poder confirmar si quiere borrar su cuenta de la base de datos.

CASO DE USO 4: Cambiar contraseña de cuenta
Clase de dominio principal: Usuario
Descripción: El usuario accede a su cuenta y se cambia su contraseña, teniendo que poner la que tiene actualmente y después confirmando la nueva que desee, actualizándose la propia base de datos.

CASO DE USO 5: Visualizar reseñas propias 
Clase de dominio principal: Vendedor
Descripción: El vendedor puede acceder a la lista de compradores que han rellenado reseñas sobre los productos del propio vendedor, pudiendo filtrar una búsqueda por los correos de los compradores.
